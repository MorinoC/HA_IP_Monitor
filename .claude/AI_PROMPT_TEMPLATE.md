# 🤖 明天AI助手的启动提示词模板

## 📋 推荐提示词 (直接复制使用)

### ✅ 标准启动提示词

```
你好！我需要继续开发 HA IP Monitor 项目。

请先按照项目规则阅读以下文件：
1. README.md
2. PROJECT_STATUS.md
3. HANDOFF_2025-11-12_v0.1.md (最新交接文件)

阅读完成后，告诉我：
1. 今天的主要任务是什么
2. 任务的优先级排序
3. 需要我确认的事项

然后我们开始今天的开发工作。
```

### 🎯 带具体任务的提示词

```

```

### 🐛 问题排查提示词

```

请先阅读项目文档了解背景：
- README.md
```

## 💡 提示词要点

### ✅ 必须包含的内容
1. **明确要求阅读3个核心文件**
   - README.md
   - PROJECT_STATUS.md
   - HANDOFF_最新日期.md

2. **明确今天的目标** (可选)
   - 具体任务
   - 优先级

3. **要求AI先理解再行动**
   - 总结任务
   - 给出计划
   - 等待确认

### ❌ 不要做的事
- ❌ 不要让AI直接开始编码
- ❌ 不要跳过阅读文档的步骤
- ❌ 不要给出模糊的任务描述

## 🔄 工作流程提醒

### AI开始工作时应该：
1. ✅ 阅读3个核心文件
2. ✅ 总结当前状态
3. ✅ 列出今日任务
4. ✅ 给出实现计划
5. ✅ 等待用户确认

### AI工作结束时应该：
1. ✅ 更新 PROJECT_STATUS.md
2. ✅ 更新 README.md
3. ✅ 删除旧交接文件
4. ✅ 创建新交接文件
5. ✅ Git commit & push

## 📞 特殊场景提示词

### 场景1: 项目暂停后重启
```
你好！HA IP Monitor项目已经暂停X天，现在要继续开发。

请详细阅读：
- README.md (了解项目)
- PROJECT_STATUS.md (了解进度)
- HANDOFF_最新日期.md (了解上次工作)

然后帮我：
1. 总结已完成的工作
2. 列出待完成的任务
3. 评估当前代码状态
4. 建议下一步计划

我们再决定从哪里开始。
```

### 场景2: 遇到技术难题
```
你好！HA IP Monitor项目遇到技术问题。

背景：请先阅读项目文档了解上下文。

问题：[详细描述问题]

需要：
1. 分析问题根源
2. 提供多个解决方案
3. 推荐最佳方案
4. 评估实现难度

请先给我分析，不要直接修改代码。
```

### 场景3: 代码审查
```
你好！请审查HA IP Monitor项目的代码。

阅读项目文档后，重点检查：
1. 代码规范 (注释是否用日语)
2. 错误处理是否完善
3. 安全问题
4. 性能优化空间
5. 文档是否同步更新

给我一份审查报告。
```
